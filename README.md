# Bank Fraud Detection — MLOps Pipeline

A end-to-end MLOps project covering data versioning, experiment tracking, and model serving.
Built as part of an MLOps course assignment using DVC, MLflow, and FastAPI.

---

```
├── data/

```

## Assignment 1 — Data Versioning with DVC

Generates a synthetic bank transaction dataset and builds a reproducible cleaning pipeline using DVC.

### What it does
- `create_data.py` generates 2000 rows with columns: `transaction_id`, `amount`, `num_transactions_24h`, `distance_from_home_km`, `is_weekend`, `is_fraud`. Fraud rate is ~5%. A small number of negative amounts are injected intentionally for the cleaning step.
- `clean_data.py` removes rows where `amount < 0` and saves the result to `data/clean.csv`.
- `dvc.yaml` defines the `clean` stage — DVC tracks the dependency between `raw.csv` and `clean.csv` and only reruns the stage when inputs change.

### How to run

```bash
# generate raw data
python create_data.py

# track with DVC
dvc add data/raw.csv
git add data/raw.csv.dvc data/.gitignore
git commit -m "track raw data"

# run the pipeline
dvc repro

# view the pipeline DAG
dvc dag

# check what changed after updating data
dvc diff
```

### dvc.yaml

```yaml
stages:
  clean:
    cmd: python clean_data.py
    deps:
      - data/raw.csv
    outs:
      - data/clean.csv
```

## Assignment 2 — Experiment Tracking with MLflow

Trains three models on the fraud dataset, logs metrics to MLflow, and automatically promotes the best model to Staging.

### Models trained
- Logistic Regression, C=0.1
- Logistic Regression, C=10.0
- Random Forest, n_estimators=50

### What is logged per run
- Parameters: model type, hyperparameters
- Metrics: AUC-ROC, AUC-PR
- Model artifact: registered under `my_fraud_detector`

### How to run

```bash
cd my-mlflow-experiments
python train.py

# launch the UI
mlflow ui --backend-store-uri sqlite:///mlflow.db
# open http://localhost:5000
```

`train.py` also calls `select_best_model()` at the end, which queries all runs, ranks by AUC-PR, and transitions the winning model version to Staging in the registry.

## Assignment 3 — Model Serving with FastAPI

Serves the trained Random Forest model as a REST API.

### Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Returns server and model status |
| GET | `/model-info` | Returns model type, version, and expected features |
| POST | `/predict` | Returns fraud probability, verdict, and risk level for one transaction |
| POST | `/predict/batch` | Same as above for a list of transactions |

### Input schema for `/predict`

```json
{
  "amount": 150.0,
  "num_transactions_24h": 3,
  "distance_from_home_km": 25.0,
  "is_weekend": 0
}
```

### Risk level mapping

| Probability | Risk Level |
|-------------|------------|
| < 0.3 | LOW |
| 0.3 – 0.7 | MEDIUM |
| > 0.7 | HIGH |

### How to run

```bash
cd my-fraud-api
uvicorn app:app --reload
# open http://localhost:8000/docs for Swagger UI
```

## Notes

- `mlflow.db` is gitignored. Run `train.py` to regenerate it.
- `data/raw.csv` is DVC-tracked. Run `create_data.py` followed by `dvc repro` to regenerate the full dataset and pipeline outputs.
- The FastAPI model is trained on startup from `data/clean.csv`. Make sure the DVC pipeline has been run before starting the server.
- MLflow UI runs on port 5000 by default. On Mac, if port 5000 is taken by AirPlay, use `mlflow ui --port 5001`.
